## クイックリファレンス

### テストピラミッド
```
        /\
       /E2E\      ← 少ない（遅い・不安定・高コスト）
      /------\
     / 統合   \   ← 中程度
    /----------\
   /ユニット   \ ← 多い（速い・安定・低コスト）
  /--------------\
```
**比率の目安**: ユニット 70% / 統合 20% / E2E 10%

### シフトレフト（早期テスト）
- 要件定義: テスタビリティを考慮
- 設計: テスト戦略を策定
- 実装: TDD（テスト駆動開発）
- テスト: 自動化で継続的実行

### テストカバレッジ目標
- **ビジネスロジック**: 80%以上
- **ユーティリティ**: 90%以上
- **UI層**: 50%以上（E2Eで補完）

### テスト技法
- **同値分割**: 入力を同値クラスに分割
- **境界値分析**: 境界値をテスト
- **デシジョンテーブル**: 条件の組み合わせ
- **状態遷移**: 状態の遷移パターン
- **ペアワイズ**: 組み合わせの削減

### テスト自動化の原則
1. **繰り返し実行**: CI/CDで自動実行
2. **高速フィードバック**: 早く失敗、早く修正
3. **信頼性**: フレキーテストの排除
4. **保守性**: テストコードも品質重視
