# 要件定義スキル

## 概要

ビジネス要件を整理し、ユーザーストーリー・受入基準を作成する専門知識。

## 主な利用エージェント

- **consultant**: ビジネス分析・要件整理
- **app-architect**: 技術的実現可能性評価
- **qa**: テスト観点での要件レビュー

## クイックリファレンス

### ユーザーストーリーの INVEST 原則
- **I**ndependent（独立している）
- **N**egotiable（交渉可能）
- **V**aluable（価値がある）
- **E**stimable（見積もり可能）
- **S**mall（小さい）
- **T**estable（テスト可能）

### 受入基準の Gherkin 形式
```gherkin
Given [前提条件]
When [操作]
Then [期待結果]
```

### 非機能要件の主要項目
1. **性能**: レスポンスタイム、スループット
2. **可用性**: 稼働率、MTTR/MTBF
3. **拡張性**: ユーザー数増加への対応
4. **セキュリティ**: 認証・認可・暗号化
5. **運用性**: 監視・ログ・バックアップ
