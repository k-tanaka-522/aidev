# フロントエンド機能拡張メモ

## 作業内容

1. **デプロイ手順の整備**
   - `deploy.sh`スクリプトの作成
   - フロントエンドデプロイ手順ドキュメントの作成
   - 環境変数の設定ファイルの作成
   
2. **AWS固有機能表示コンポーネントの設計**
   - AWSPricingDisplay: AWS料金計算結果を表形式で表示
   - AWSArchitectureDisplay: インフラ構成図を視覚的に表示
   - AiMessageParser: AIレスポンスを解析して特殊フォーマットを認識
   
3. **次のステップ**
   - AWS固有コンポーネントの実装
   - エージェント選択UI機能の拡張
   - ブランディング要素の適用

## 設計メモ

### AWS固有機能の表示方法

AIアシスタントのレスポンス内に含まれる特殊なフォーマットを検出し、適切なコンポーネントでレンダリングする方法を採用します：

1. **料金情報フォーマット**
```
```aws-pricing
{
  "totalCost": 1234.56,
  "currency": "USD",
  "services": [
    { "name": "EC2", "cost": 500.00, "details": "..." },
    { "name": "RDS", "cost": 300.00, "details": "..." }
  ],
  "options": [
    { "name": "予約インスタンス適用", "savings": 200.00 }
  ]
}
```
```

2. **構成図情報フォーマット**
```
```aws-architecture
{
  "title": "3層アーキテクチャ構成",
  "description": "耐障害性と拡張性を考慮したウェブアプリケーション構成",
  "components": [
    {
      "name": "VPC",
      "type": "vpc",
      "details": "CIDR: 10.0.0.0/16"
    },
    {
      "name": "Webサーバー",
      "type": "ec2",
      "parent": "VPC",
      "details": "t3.medium, Auto Scaling: 2-5台"
    }
  ]
}
```
```

### 実装の優先順位

1. まずAiMessageParserを実装してAIレスポンスの解析機能を提供
2. 次にAWSPricingDisplayコンポーネントを実装（料金情報表示が最優先）
3. 最後にAWSArchitectureDisplayコンポーネントを実装

### エージェント選択UI拡張のアイデア

1. より直感的なエージェント選択インターフェース
   - エージェントのアイコンと簡単な説明を含むカード形式
   - エージェントの専門分野や得意な質問タイプを表示
   
2. エージェント状態の可視化
   - 現在選択中のエージェントを強調表示
   - エージェント間の連携状態を表示（連携中のエージェント名など）
   - エージェント切り替え履歴の表示
