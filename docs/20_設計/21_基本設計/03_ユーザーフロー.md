# aiDev ユーザーフロー

## 1. ユーザーフロー概要

aiDevシステムにおけるユーザーフローは、ユーザーがAIエージェントとの対話を通じて開発支援を受けるプロセスを定義します。このフローは、ユーザーの目的に応じて柔軟に分岐し、適切なエージェントとの対話を実現します。

## 2. 主要ユーザーフロー

### 2.1 初回利用フロー

1. **アカウント登録・ログイン**
   - ユーザーがシステムにアクセス
   - Cognitoを使用した認証
   - 初回ユーザー情報登録

2. **プロジェクト作成**
   - 新規プロジェクト情報入力
   - 業種・規模などの基本情報設定
   - 開発の目的・ゴール設定

3. **初期ヒアリング**
   - 営業・プリセールスエージェントによる初期対話
   - ニーズと課題の把握
   - プロジェクト目標の明確化

4. **支援プランの提示**
   - 想定される開発工程と支援内容の提案
   - 必要なリソースと期間の概算
   - 期待される成果の提示

### 2.2 開発相談フロー

1. **プロジェクト選択**
   - 既存プロジェクトの一覧表示
   - プロジェクト選択または新規作成

2. **相談内容の入力**
   - 相談したい内容の自然言語による入力
   - 関連資料のアップロード（オプション）

3. **エージェント選択**
   - システムが適切なエージェントを推奨
   - ユーザーによるエージェント選択（オプション）

4. **対話セッション**
   - 選択されたエージェントとの対話
   - 追加質問によるニーズの明確化
   - 解決策の生成と提示

5. **成果物の生成・保存**
   - 対話結果に基づく成果物の生成
   - プロジェクトへの保存
   - 成果物の編集（必要に応じて）

### 2.3 AWS環境構築フロー

1. **要件入力**
   - 構築したい環境の要件を自然言語で入力
   - 既存環境情報の入力（あれば）

2. **環境設計**
   - システムアーキテクトエージェントによる設計案生成
   - 設計案のレビューと調整
   - 最終設計の確定

3. **IaCテンプレート生成**
   - CloudFormationテンプレートの自動生成
   - テンプレートのレビューと編集（オプション）

4. **環境構築実行**
   - 生成されたテンプレートによる環境構築
   - 構築状況のリアルタイム表示
   - 完了通知と接続情報の提供

5. **構築後検証**
   - 環境の正常性チェック
   - 設定値の確認
   - 動作テスト結果のレポート

### 2.4 エージェント連携フロー

1. **対話中のエージェント切替**
   - 現在のエージェントが他のエージェントの専門性が必要と判断
   - ユーザーに切替を提案
   - 承認後、コンテキストを保持したまま切替

2. **マルチエージェントセッション**
   - 複数エージェントが同時に対話に参加
   - 各エージェントがそれぞれの専門領域から意見
   - 総合的な解決策の提示

3. **クロスレビュー**
   - あるエージェントの成果物を別のエージェントがレビュー
   - 改善点の指摘と修正提案
   - 修正後の最終成果物の生成

## 3. 状態遷移

### 3.1 プロジェクト状態

- **計画中**: 初期段階、要件収集中
- **設計中**: アーキテクチャや詳細設計を作成中
- **実装中**: コード作成やインフラ構築中
- **テスト中**: 動作検証や品質確認中
- **完了**: すべての作業が完了
- **保守中**: リリース後の運用・改善フェーズ

### 3.2 セッション状態

- **開始**: セッション開始時
- **ヒアリング中**: ユーザーのニーズ把握中
- **提案中**: 解決策の提案中
- **確認中**: ユーザーからの承認待ち
- **実行中**: 承認された作業の実行中
- **完了**: セッション完了
- **一時中断**: 中断状態（後で再開可能）

## 4. 画面遷移の基本フロー

### 4.1 認証フロー
1. **ログイン画面** → **ダッシュボード**

### 4.2 プロジェクト管理フロー
1. **ダッシュボード** → **プロジェクト一覧** → **プロジェクト詳細**
2. **ダッシュボード** → **プロジェクト作成** → **プロジェクト詳細**

### 4.3 対話フロー
1. **プロジェクト詳細** → **対話セッション**
2. **対話セッション** → **成果物詳細**

### 4.4 環境構築フロー
1. **プロジェクト詳細** → **環境設計** → **構築実行** → **環境管理**

## 5. エラーハンドリングの基本フロー

### 5.1 入力エラー
- 不明確な入力に対する明確化要求
- 情報不足時の追加情報リクエスト

### 5.2 処理エラー
- AI処理エラー時のフォールバック応答
- 要求制限到達時の通知とキュー処理

### 5.3 構築エラー
- 環境構築失敗時の詳細エラー情報表示
- 修正提案と再試行オプション

## 6. フェーズ別実装

### 6.1 フェーズ1（MVP）
- 基本的な対話フロー
- シンプルなプロジェクト管理
- 初期ヒアリングと提案機能

### 6.2 フェーズ2
- エージェント切替フロー
- 環境構築フローの完全実装
- 高度なエラーハンドリング

### 6.3 フェーズ3
- マルチエージェントセッション
- クロスレビューフロー
- 高度な状態管理と継続的支援
