# aiDev データモデル

## 1. データモデル概要

aiDevシステムでは、以下の主要データエンティティを定義し、それらの関係とデータフローを設計します。このデータモデルは、複数のAIエージェントが協調してユーザーを支援するための基盤となります。

## 2. 主要エンティティ

### 2.1 ユーザー (User)

ユーザー情報を管理するエンティティです。

**主な属性**:
- ユーザーID (PK)
- メールアドレス
- 名前
- 組織
- 役割/職種
- 設定/環境設定
- 認証情報（Cognitoと連携）

### 2.2 プロジェクト (Project)

ユーザーが取り組む開発プロジェクトを表すエンティティです。

**主な属性**:
- プロジェクトID (PK)
- プロジェクト名
- 説明
- 作成日時
- 最終更新日時
- ステータス（計画中/設計中/実装中/テスト中/完了/保守中）
- オーナーID (ユーザーへの参照)
- タグ/カテゴリ

### 2.3 セッション (Session)

ユーザーとAIエージェントの対話セッションを表すエンティティです。

**主な属性**:
- セッションID (PK)
- プロジェクトID (FK)
- 開始日時
- 終了日時
- セッションタイプ（ヒアリング/設計/コーディング/レビュー等）
- ステータス（進行中/完了/一時停止）
- 概要/タイトル

### 2.4 メッセージ (Message)

セッション内の個々のメッセージを表すエンティティです。

**主な属性**:
- メッセージID (PK)
- セッションID (FK)
- タイムスタンプ
- 送信者（ユーザー/エージェント種別）
- メッセージ内容
- メッセージタイプ（テキスト/コード/図表等）
- 参照リソース（URL/ファイルID等）

### 2.5 エージェント (Agent)

システム内の各AIエージェントを表すエンティティです。

**主な属性**:
- エージェントID (PK)
- エージェント名
- 役割/タイプ
- 説明
- 使用モデル（Claude/Hermes等）
- スキル/専門分野のタグ
- アクティブフラグ

### 2.6 成果物 (Artifact)

対話を通じて生成された成果物を表すエンティティです。

**主な属性**:
- 成果物ID (PK)
- プロジェクトID (FK)
- セッションID (FK)
- 成果物タイプ（ドキュメント/コード/図表等）
- タイトル
- 作成日時
- 最終更新日時
- 内容/本文
- 生成エージェントID
- レビュー状態
- バージョン
- S3パス（大きなファイルの場合）

### 2.7 環境 (Environment)

AIが設計・構築するAWS環境を表すエンティティです。

**主な属性**:
- 環境ID (PK)
- プロジェクトID (FK)
- 環境名
- 環境タイプ（開発/テスト/本番等）
- ステータス（設計中/構築中/稼働中/停止等）
- 作成日時
- インフラ構成（JSON形式）
- アクセス情報
- 関連リソース（CloudFormationスタック等）

### 2.8 ナレッジベース (KnowledgeBase)

エージェントが参照する専門知識を表すエンティティです。

**主な属性**:
- ナレッジID (PK)
- カテゴリ
- タイトル
- 内容
- 作成日時
- 最終更新日時
- タグ
- ソース/出典
- 信頼性スコア

## 3. エンティティ関連図

```
User 1──*┐
         │
         v
       Project 1──*┐
                   │
                   ├──► Session 1──* Message
                   │
                   ├──► Artifact
                   │
                   └──► Environment
                   
Agent *──* Session

KnowledgeBase *──* Agent
```

## 4. データストレージ設計

### 4.1 DynamoDB テーブル

#### Users テーブル
- パーティションキー: userId
- GSI: email-index

#### Projects テーブル
- パーティションキー: projectId
- GSI: userId-createdAt-index

#### Sessions テーブル
- パーティションキー: sessionId
- ソートキー: createdAt
- GSI: projectId-createdAt-index

#### Messages テーブル
- パーティションキー: sessionId
- ソートキー: timestamp
- GSI: messageId-index

#### Artifacts テーブル
- パーティションキー: artifactId
- GSI: projectId-createdAt-index
- GSI: sessionId-createdAt-index

#### Environments テーブル
- パーティションキー: environmentId
- GSI: projectId-status-index

### 4.2 S3 バケット

#### Knowledge Bucket
- ナレッジベースのコンテンツ
- ドキュメント、画像、動画等

#### Artifact Bucket
- 大型の成果物ファイル
- コードアーカイブ、設計図等

#### Template Bucket
- CloudFormationテンプレート
- 環境構築スクリプト

### 4.3 ElasticSearch (オプション)
- ナレッジベースの全文検索
- 過去の対話履歴の意味検索

## 5. データフロー

### 5.1 ユーザー対話データフロー
1. ユーザーがメッセージを送信
2. メッセージテーブルに保存
3. 関連セッション情報の更新
4. AIエージェントによる処理
5. 応答メッセージの生成と保存
6. 必要に応じて成果物の生成と保存

### 5.2 環境構築データフロー
1. ユーザーが環境構築を要求
2. 環境エンティティの作成（設計中状態）
3. IaCテンプレートの生成とS3保存
4. CloudFormationによる構築実行
5. 構築結果の環境エンティティへの反映

### 5.3 ナレッジベース参照フロー
1. AIエージェントが処理中に関連知識を検索
2. ElasticSearchによる関連コンテンツ検索
3. 検索結果をAIプロンプトに統合
4. 知識に基づく応答生成

## 6. データ保持ポリシー

- ユーザーデータ: アカウント削除まで保持
- プロジェクトデータ: プロジェクト完了後3年間保持
- セッション/メッセージ: プロジェクト完了後1年間保持
- 未使用環境情報: 作成後6ヶ月未使用の場合アーカイブ
- アーカイブデータ: S3 Glacierに移動（コスト最適化）

## 7. フェーズ別実装計画

### 7.1 フェーズ1（MVP）
- 基本的なユーザー、プロジェクト、セッション、メッセージテーブル
- シンプルな成果物管理
- 基本的なナレッジベース構造

### 7.2 フェーズ2
- 環境テーブルの完全実装
- 拡張ナレッジベース
- 成果物バージョン管理

### 7.3 フェーズ3
- エージェント間連携データモデルの拡張
- 高度な検索・分析機能
- 機械学習によるパーソナライズ
