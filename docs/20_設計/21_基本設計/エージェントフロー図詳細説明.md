# aiDev エージェントフロー図詳細説明

## 1. 概要

本ドキュメントでは、aiDevシステムにおける複数のAIエージェント間の連携フローを詳細に説明します。システム内の各エージェントの役割、連携パターン、フェーズ別の実装計画について視覚的に整理しています。

## 2. エージェントグループ

aiDevシステムのエージェントは、以下の6つの機能グループに分類されています：

### 2.1 プリセールスエージェント群
- **PS01**: プリセールスリード - 顧客との初期接点、要件ヒアリング、全体提案調整
- **PS02**: AWSソリューション提案 - AWS技術提案の詳細化
- **PS03**: コスト最適化専門 - コスト見積もり、最適化提案

### 2.2 ITコンサルタントエージェント群
- **IT01**: IT戦略リード - 事業戦略とIT戦略の整合性分析
- **IT02**: DXコンサルタント - デジタル変革推進支援
- **IT03**: 業界特化コンサルタント - 業界別ベストプラクティス提案

### 2.3 システムアーキテクトエージェント群
- **SA01**: アーキテクチャリード - システム全体設計
- **SA02**: セキュリティアーキテクト - セキュリティ設計・対策
- **SA03**: データアーキテクト - データモデル・フロー設計

### 2.4 プロジェクトマネージャーエージェント群
- **PM01**: プロジェクトリード - 計画策定、進捗管理
- **PM02**: アジャイルコーチ - アジャイル開発支援
- **PM03**: リスク管理 - リスク特定、対策立案

### 2.5 開発エンジニアエージェント群
- **DEV01**: フロントエンド開発 - UI/UX実装
- **DEV02**: バックエンド開発 - API、ビジネスロジック実装
- **DEV03**: インフラ開発 - IaC実装、クラウド環境構築

### 2.6 QAエンジニアエージェント群
- **QA01**: テストリード - テスト計画、設計
- **QA02**: 自動化テスト - テスト自動化
- **QA03**: セキュリティテスト - 脆弱性検証

## 3. 連携パターン

エージェント間の連携は、以下の3つの基本パターンで行われます：

### 3.1 リレー型連携
一つのエージェントから別のエージェントへタスクを引き継ぐパターン。各エージェントは自身の専門領域の処理を完了した後、次のエージェントにコンテキストとともにタスクを委譲します。

**例**: PS01(プリセールスリード) → IT01(IT戦略リード) → PS01(プリセールスリード)

### 3.2 並行型連携
複数のエージェントが並行して処理を行い、その結果を統合するパターン。マスターエージェントがタスクを分解し、各エージェントに割り当て、結果を収集・統合します。

**例**: SA01(アーキテクチャリード) → [SA02(セキュリティ) + SA03(データ) + DEV03(インフラ)] → SA01(結果統合)

### 3.3 階層型連携
上位エージェントが下位エージェントを監督・指示するパターン。上位エージェントは作業計画や方針を決定し、下位エージェントは実行と報告を担当します。

**例**: PM01(プロジェクトリード) → QA01(テストリード) → [QA02(自動化) + QA03(セキュリティ)]

## 4. 主要ワークフロー

aiDevシステムにおける主要なワークフローは以下の通りです：

### 4.1 初期相談〜提案フロー
1. ユーザーからの初期相談をPS01(プリセールスリード)が受け、基本要件をヒアリング
2. IT01(IT戦略リード)に連携し、事業戦略とIT整合性の分析、長期的方針提案を取得
3. PS02(AWS提案)とPS03(コスト最適化)と連携し、具体的な提案と見積を作成
4. SA01(アーキテクト)と連携し、技術的な実現性を確認
5. PS01がユーザーに統合された提案内容を提示

### 4.2 AWS環境構築フロー
1. PS01が環境要件を詳細化し、SA01(アーキテクト)に連携
2. SA01が全体アーキテクチャを設計し、SA02(セキュリティ)と連携
3. PM01(プロジェクト管理)が計画を策定、リスク評価をPM03と実施
4. DEV03(インフラ開発)がCloudFormationテンプレート作成
5. QA01(テスト)がテスト計画を策定し実行
6. 最終確認後、PS01がユーザーに環境を納品・説明

### 4.3 アプリケーション開発フロー
1. PM01が要件管理とスプリント計画を策定
2. SA01が設計を主導、SA02/SA03と連携
3. DEV01(フロントエンド)とDEV02(バックエンド)が並行実装
4. QA01/QA02/QA03がテストを実施
5. PM01が納品準備とリリース管理

## 5. フェーズ別実装計画

エージェントの実装は3フェーズに分けて段階的に行います：

### 5.1 フェーズ1（MVP）
- **対象エージェント**: PS01, IT01, SA01
- **基本連携**: リレー型連携の基本実装
- **主要機能**: 初期ヒアリング、AWS提案、基本アーキテクチャ設計

### 5.2 フェーズ2
- **追加エージェント**: PS02, PS03, SA02, PM01, DEV03
- **追加連携**: 並行型連携、一部の階層型連携
- **追加機能**: コスト最適化、セキュリティ設計、プロジェクト管理、IaC実装

### 5.3 フェーズ3
- **追加エージェント**: IT02, IT03, PM02, PM03, SA03, DEV01, DEV02, QA01, QA02, QA03
- **拡張連携**: 完全な階層型連携、高度な並行処理
- **追加機能**: 業界特化コンサルティング、アジャイル管理、フルスタック開発、品質保証

## 6. エージェント間メッセージ

エージェント間の通信には、目的に応じた以下のメッセージタイプを使用します：

- **TASK_HANDOFF**: リレー型連携でのタスク引き継ぎ
- **TASK_ACCEPTED**: ハンドオフされたタスクの受領確認
- **TASK_DISTRIBUTE**: 並行型連携でのタスク配布
- **TASK_RESULT**: 処理結果の提出
- **RESULT_INTEGRATION**: 統合結果の共有
- **TASK_DELEGATE**: 階層型連携でのタスク委任
- **PROGRESS_REPORT**: 進捗報告
- **TASK_COMPLETE**: タスク完了報告

## 7. 連携フロー図の見方

添付のエージェントフロー図は以下の情報を視覚化しています：

- **ノードの形状**: 楕円形はエージェント、人型はユーザー
- **ノードの色**: 各エージェントグループごとに色分け
- **ノードの線の太さ**: フェーズ1は太線、フェーズ2は中線、フェーズ3は細線
- **ノードの透明度**: フェーズ3は半透明で表示
- **矢印の種類**: 
  - 双方向矢印：リレー型連携（相互やりとり）
  - 単方向矢印：階層型連携（指示・報告）
  - 点線矢印：イベント駆動型連携
- **矢印のラベル**: 連携パターンや特殊な関係を表示
