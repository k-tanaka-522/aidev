# aiDev システム概要

## 1. システムの目的

aiDevシステムは、「専門業者に依頼していたクリエイティブ制作をAIとの対話だけで実現する」という未来を目指す第一歩として、AWS Bedrockを活用した「AI開発支援窓口（プリセールスエージェント）」を構築するプロジェクトです。

このシステムは複数のAIエージェントが協調し、システム開発の各工程（企画、要件定義、設計、実装、テスト、運用）をサポートします。ユーザーは自然言語による対話を通じて、AIエージェントの専門知識を活用できます。

## 2. システム全体像

aiDevシステムは、以下の主要コンポーネントで構成されます：

1. **フロントエンド**: ユーザーとAIエージェントの対話インターフェース
2. **エージェントオーケストレーター**: 複数エージェントの管理と連携
3. **AIエージェント群**: 各専門分野に特化したAIエージェント
4. **ナレッジベース**: エージェントが参照する専門知識
5. **プロジェクト管理**: プロジェクト状態やコンテキストの管理
6. **環境構築自動化**: AWS環境の自動設計・構築機能
7. **AWS Organizations連携**: 顧客ごとの子アカウント自動作成と管理

## 3. 主要機能と特徴

### 3.1 複数エージェントによる協調開発支援

- システム開発の各工程に対応した専門エージェント
- エージェント間の連携によるシームレスな開発サポート
- クロスレビューによる品質向上

### 3.2 ナレッジベースを活用した専門的支援

- AWS設計・構築に関する専門知識
- 業界別のベストプラクティス
- 最新の技術トレンドと実装パターン

### 3.3 対話型インターフェース

- 自然言語によるやり取り
- プロジェクトコンテキストの保持
- 生成コンテンツの可視化と編集

### 3.4 環境構築自動化

- AWSリソースの自動設計・構築
- IaCテンプレートの生成
- 環境の発行と管理

### 3.5 AWS Organizations連携

- メールアドレスのみの簡易登録による子アカウント自動作成
- 顧客ごとに分離された子アカウントでのプリセールスエージェント実行
- クロスアカウント認証によるシームレスなアクセス提供
- 親アカウントによる一元管理と監視

## 4. 開発フェーズ

システムは段階的に開発され、各フェーズで機能を拡充していきます：

1. **フェーズ1（MVP）**: プリセールス特化エージェントの実装
2. **フェーズ2**: 実装支援機能の追加
3. **フェーズ3**: 持続的関係構築機能の追加

## 5. 技術スタック

- **フロントエンド**: React, Next.js, TypeScript
- **バックエンド**: AWS Lambda, API Gateway, Node.js
- **AI基盤**: AWS Bedrock, Claude 3.7/3.5, Nous Hermes
- **データストア**: DynamoDB, S3
- **認証**: Amazon Cognito
- **インフラ**: CloudFormation, CloudFront

## 6. システム要件

### 6.1 パフォーマンス要件

- API応答時間: 500ms以内（AI生成時間を除く）
- AI応答生成時間: 平均5秒以内
- 同時接続ユーザー: 最大100ユーザー

### 6.2 セキュリティ要件

- ユーザー認証
- データ暗号化（保存時・転送時）
- 適切なアクセス制御

### 6.3 可用性要件

- 稼働率: 99.9%以上
- バックアップ: 日次
- 障害復旧時間: 1時間以内

## 7. 前提条件と制約

- AWS Bedrockの利用制限内での運用
- コンテキスト長の制限に対応
- 対応ブラウザ: Chrome, Firefox, Safari, Edge最新版
