# aiDev システム概要

## 1. システムの目的

aiDevシステムは、「専門業者に依頼していたクリエイティブ制作をAIとの対話だけで実現する」という未来を目指す第一歩として、AWS Bedrockを活用した「AI開発支援窓口（プリセールスエージェント）」を構築するプロジェクトです。

このシステムは複数のAIエージェントが協調し、システム開発の各工程（企画、要件定義、設計、実装、テスト、運用）をサポートします。ユーザーは自然言語による対話を通じて、AIエージェントの専門知識を活用できます。

## 2. システム全体像

aiDevシステムは、以下の主要コンポーネントで構成されます：

1. **フロントエンド**: ユーザーとAIエージェントの対話インターフェース
2. **エージェントオーケストレーター**: 複数エージェントの管理と連携
3. **AIエージェント群**: 各専門分野に特化したAIエージェント
4. **ナレッジベース**: エージェントが参照する専門知識
5. **プロジェクト管理**: プロジェクト状態やコンテキストの管理
6. **環境構築自動化**: AWS環境の自動設計・構築機能
7. **AWS Organizations連携**: 顧客ごとの子アカウント自動作成と管理

## 3. 主要機能と特徴

### 3.1 複数エージェントによる協調開発支援

- システム開発の各工程に対応した専門エージェント
- エージェント間の連携によるシームレスな開発サポート
- クロスレビューによる品質向上

### 3.2 ナレッジベースを活用した専門的支援

- AWS設計・構築に関する専門知識
- 業界別のベストプラクティス
- 最新の技術トレンドと実装パターン

### 3.3 対話型インターフェース

- 自然言語によるやり取り
- プロジェクトコンテキストの保持
- 生成コンテンツの可視化と編集

### 3.4 環境構築自動化

- AWSリソースの自動設計・構築
- IaCテンプレートの生成
- 環境の発行と管理

### 3.5 AWS Organizations連携

- メールアドレスのみの簡易登録による子アカウント自動作成
- 顧客ごとに分離された子アカウントでのプリセールスエージェント実行
- クロスアカウント認証によるシームレスなアクセス提供
- 親アカウントによる一元管理と監視

### 3.6 AIアウトプット管理

- **生成コンテンツ**
  - 設計ドキュメント（要件定義書、設計書、運用手順書等）
  - システム構成図（クラウド/オンプレミス、ネットワーク図、システム連携図等）
  - IaCコード（CloudFormation/CDK/Terraform等）
  - アプリケーションコード（サンプル実装、テストコード等）
  - 実行形式ファイル（設定済み環境のエクスポート等）

- **視覚化と編集**
  - JSON/YAMLベースの中間フォーマットによる構造化データ出力
  - Draw.io互換形式でのシステム構成図出力と視覚化
  - インタラクティブな構成図エディタによる編集・カスタマイズ

- **統合出力管理**
  - S3バケットへの一元的なコンテンツ保存（ナレッジベースとの統合）
  - プロジェクト単位でのバージョン管理と継続的な相談履歴との紐付け
  - 差分比較と変更履歴トラッキング
  - メタデータによる高度な検索・フィルタリング

- **外部連携（将来拡張）**
  - MCP経由でのSaaS連携（Notion、GitHub、Confluence等）
  - チーム共同編集のためのリアルタイム同期
  - CI/CDパイプラインとの統合
  - 企業内ナレッジベースへの自動取り込み

## 4. 開発フェーズ

システムは段階的に開発され、各フェーズで機能を拡充していきます：

### フェーズ1：プリセールス特化（3ヶ月）
- 質問応答機能の実装
- ナレッジベースの基本構築
- 簡易設計提案機能
- 判断基準：月間対話数100件達成、商談化率15%以上

### フェーズ2：実装支援機能追加（+3ヶ月）
- 環境構築自動化機能の本格実装
- より詳細な設計支援機能
- テンプレートライブラリの拡充
- 判断基準：自動化機能利用率30%以上、提案満足度4.0/5.0以上

### フェーズ3：持続的関係構築（+6ヶ月）
- 環境発行機能の実装
- 運用監視提案機能
- 最適化提案機能の追加
- 判断基準：リピート率70%以上、SES展開率20%以上

## 5. 技術スタック

- **フロントエンド**: React, Next.js, TypeScript
- **バックエンド**: AWS Lambda, API Gateway, Node.js
- **AI基盤**: AWS Bedrock, Claude 3.7/3.5, Nous Hermes
- **データストア**: DynamoDB, S3
- **認証**: Amazon Cognito
- **インフラ**: CloudFormation, CloudFront

## 6. システム要件

### 6.1 パフォーマンス要件

- API応答時間: 500ms以内（AI生成時間を除く）
- AI応答生成時間: 平均5秒以内
- 同時接続ユーザー: 最大100ユーザー

### 6.2 セキュリティ要件

- ユーザー認証
- データ暗号化（保存時・転送時）
- 適切なアクセス制御

### 6.3 可用性要件

- 稼働率: 99.9%以上
- バックアップ: 日次
- 障害復旧時間: 1時間以内

## 7. 前提条件と制約

- AWS Bedrockの利用制限内での運用
- コンテキスト長の制限に対応
- 対応ブラウザ: Chrome, Firefox, Safari, Edge最新版
