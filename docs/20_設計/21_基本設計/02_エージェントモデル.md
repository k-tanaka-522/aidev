# aiDev エージェントモデル

## 1. エージェントモデルの概要

aiDevシステムは、システム開発の各工程に対応した専門的なAIエージェントによる協調型の開発支援を提供します。各エージェントは特定の役割と専門知識を持ち、ユーザーの目的に応じて適切なエージェントが対応します。

## 2. エージェント種類と役割

### 2.1 営業・プリセールスエージェント

**役割**: 
- 顧客ニーズの把握
- 要件の初期ヒアリング
- ROI提案と予算化支援
- 提案資料の作成支援

**特徴**:
- ビジネス視点からの提案能力
- コスト効果分析能力
- わかりやすい説明と資料作成能力
- 投資対効果の最大化提案

**使用モデル**: Claude 3.7/3.5 Sonnet

### 2.2 ITコンサルタント/ストラテジストエージェント

**役割**:
- 事業戦略とITの整合性確保
- デジタルトランスフォーメーション支援
- 長期的な技術戦略の提案

**特徴**:
- 業界知識と技術トレンド分析
- 経営視点からのIT戦略提案
- 中長期ロードマップ策定能力

**使用モデル**: Claude 3.7 Sonnet

### 2.3 プロジェクトマネージャーエージェント

**役割**:
- 要件管理とスコープ定義
- スケジュール・リソース管理
- 進捗状況の監視と報告

**特徴**:
- プロジェクト全体を俯瞰する能力
- リスク管理と課題解決能力
- コミュニケーション調整能力

**使用モデル**: Claude 3.5 Sonnet

### 2.4 システムアーキテクトエージェント

**役割**:
- システム全体設計
- 技術選定と構成設計
- 非機能要件の設計への反映

**特徴**:
- 広範な技術知識
- アーキテクチャパターンの応用
- トレードオフの分析と最適解提案

**使用モデル**: Claude 3.7 Sonnet

### 2.5 開発エンジニア(SE/PG)エージェント

**役割**:
- 詳細設計
- コーディング支援
- 単体テスト設計

**特徴**:
- 実装知識とベストプラクティス
- コード生成とレビュー能力
- 問題解決能力

**使用モデル**: Nous Hermes Pro / Claude 3 Haiku

### 2.6 QAエンジニアエージェント

**役割**:
- テスト計画と設計
- テストケース作成
- 品質評価

**特徴**:
- 網羅的検証思考
- バグ発見能力
- 品質メトリクス分析

**使用モデル**: Nous Hermes / Claude 3 Haiku

## 3. エージェント連携モデル

### 3.1 連携の基本原則

- **ワークフロー基本形**: 前工程のアウトプットを次工程のインプットとする
- **横断的レビュー**: 複数エージェントによるクロスチェック
- **コンテキスト共有**: プロジェクト情報の一貫した共有
- **エスカレーション**: 判断困難な場合は上位エージェントへエスカレーション

### 3.2 連携パターン

1. **リレー型連携**:
   - 工程ごとにエージェントをリレーで切り替え
   - 各工程の成果物を次工程に引き継ぐ

2. **並行型連携**:
   - 複数エージェントが並行して作業
   - 結果を統合して最終成果物を作成

3. **階層型連携**:
   - 上位エージェントが下位エージェントに作業を委任
   - 上位エージェントが結果を確認・統合

### 3.3 エージェント間コミュニケーション

- **メッセージ形式**: 構造化フォーマット（JSON）
- **参照情報**: プロジェクトコンテキスト、過去の対話履歴
- **メタデータ**: 信頼度、推論根拠、参照情報

## 4. エージェント状態モデル

### 4.1 基本状態

- **待機**: ユーザー入力または他エージェントからの入力待ち
- **処理中**: 入力に対する処理を実行中
- **レビュー中**: 他エージェントからのレビューを行っている
- **応答準備完了**: 応答の準備が完了
- **エラー**: 何らかのエラーが発生

### 4.2 状態遷移の基本フロー

1. 待機 → 処理中: 入力受信時
2. 処理中 → レビュー中: 処理完了後、必要に応じて
3. レビュー中 → 処理中: レビュー結果に基づく修正が必要な場合
4. 処理中/レビュー中 → 応答準備完了: 最終結果の準備完了
5. 任意の状態 → エラー: エラー発生時
6. エラー → 待機: エラー解決後

## 5. エージェント拡張性

- 新たな専門エージェントの追加が可能な設計
- モデル差し替えに対応した抽象化レイヤー
- エージェント定義ファイルによる宣言的設定
