# aiDev エージェントID定義

## 1. 概要

本ドキュメントでは、aiDevシステムで使用する全エージェントのID、基本属性、および役割の定義を行います。この定義はシステム全体で統一的に参照され、エージェント間連携の基盤となります。

## 2. エージェントID体系

エージェントIDは、以下の命名規則に従います：

`AGENT_[役割略称][連番]`

例：`AGENT_PS01` (プリセールス第1エージェント)

## 4. エージェント属性定義

各エージェントは以下の属性を持ちます：

```json
{
  "agent_id": "AGENT_PS01",
  "name": "プリセールスリード",
  "type": "プリセールス",
  "description": "初期ヒアリングからAWS提案まで一貫して担当するメインエージェント",
  "model": "claude-3-7-sonnet",
  "skills": ["初期ヒアリング", "AWS提案", "コスト試算", "アーキテクチャ提案"],
  "priority": "high",
  "system_prompt_template": "system_prompts/ps01_template.txt",
  "can_collaborate_with": ["AGENT_PS02", "AGENT_PS03", "AGENT_IT01", "AGENT_SA01"],
  "handoff_triggers": [
    {
      "trigger_pattern": "IT戦略の詳細相談",
      "target_agent": "AGENT_IT01",
      "context_transfer": ["project_goals", "business_context"]
    },
    {
      "trigger_pattern": "詳細なアーキテクチャ設計",
      "target_agent": "AGENT_SA01", 
      "context_transfer": ["requirements", "aws_services"]
    }
  ],
  "active": true
}
```

## 5. フェーズ別実装計画

### 5.1 フェーズ1（MVP）

フェーズ1では、以下のエージェントを優先的に実装します：

- AGENT_PS01：プリセールスリード
- AGENT_IT01：IT戦略リード
- AGENT_SA01：アーキテクチャリード（基本機能のみ）

### 5.2 フェーズ2

フェーズ2では、以下のエージェントを追加実装します：

- AGENT_PS02：AWSソリューション提案
- AGENT_PS03：コスト最適化専門
- AGENT_PM01：プロジェクトリード
- AGENT_SA02：セキュリティアーキテクト
- AGENT_DEV03：インフラ開発

### 5.3 フェーズ3

フェーズ3では、残りのエージェントを必要に応じて実装します：

- AGENT_IT02：DXコンサルタント
- AGENT_IT03：業界特化コンサルタント
- AGENT_PM02：アジャイルコーチ
- AGENT_PM03：リスク管理
- AGENT_SA03：データアーキテクト
- AGENT_DEV01：フロントエンド開発
- AGENT_DEV02：バックエンド開発
- AGENT_QA01：テストリード
- AGENT_QA02：自動化テスト
- AGENT_QA03：セキュリティテスト

## 6. エージェント拡張性

新たなエージェントを追加する際は、以下の手順で行います：

1. 本定義ファイルに新エージェントのID・属性を追加
2. エージェント詳細定義ドキュメントの作成
3. システムプロンプトテンプレートの作成
4. エージェント連携プロトコルの更新
5. 新エージェントのテスト・検証
6. 本番環境への反映

エージェントの追加・変更はバージョン管理され、変更履歴が記録されます。
