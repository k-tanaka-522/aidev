# ITコンサルタントエージェント詳細設計

## 1. 概要

ITコンサルタントエージェントは、事業戦略とIT戦略の整合性を図り、デジタルトランスフォーメーションを支援するAIエージェントです。ビジネス目標達成のためのIT戦略立案、技術選定、ROI分析などを通じて、経営視点からの包括的なアドバイスを提供します。業界知識と技術トレンドを組み合わせ、中長期的な視点でのIT投資計画や技術ロードマップの策定を支援します。

## 2. 役割と機能

### 2.1 主要な役割

1. **IT戦略立案**
   - 事業戦略とITの整合性分析
   - 中長期IT投資計画の策定支援
   - デジタル化優先領域の特定

2. **デジタルトランスフォーメーション推進**
   - DX成熟度評価
   - 組織・プロセス変革の提案
   - デジタル化による事業創出支援

3. **技術動向分析**
   - 業界技術トレンド予測
   - 競合他社の技術動向調査
   - 新技術の事業インパクト評価

4. **投資対効果分析**
   - IT投資のROI試算
   - コスト削減ポイントの特定
   - 投資優先順位付け

5. **課題解決提案**
   - ITに関する経営課題の特定
   - ソリューションアプローチの提案
   - ベストプラクティスの提示

## 3. インプット・アウトプット

### 3.1 インプット情報

#### 3.1.1 ユーザーからの入力

- 事業計画や経営戦略情報
- 現状のIT環境・課題
- 市場環境や競合状況
- 投資予算や制約条件
- 組織構造や企業文化情報

#### 3.1.2 システムからの入力

- ユーザープロファイル情報

- 業界データと市場動向
- 技術トレンド情報
- プリセールスエージェントからの引継ぎ情報

### 3.2 アウトプット情報

#### 3.2.1 基本レスポンス

- 戦略的アドバイス
- 課題解決提案
- 質問と確認

#### 3.2.2 生成コンテンツ

- IT戦略ロードマップ
- デジタル成熟度評価レポート
- 投資対効果分析
- 技術トレンドサマリー
- アクションプラン

## 4. プロンプト設計

### 4.1 システムプロンプト

```
あなたはITコンサルタント・ストラテジストとして、企業のデジタル戦略立案と実行を支援します。以下の役割と責務を持ちます：

1. 事業戦略とIT戦略の整合性を分析し、最適な方向性を提案する
2. デジタルトランスフォーメーションの推進に向けた具体的なアドバイスを提供する
3. 業界トレンドと新技術の事業インパクトを評価する
4. IT投資の優先順位付けとROI分析を支援する
5. 中長期的な技術ロードマップを策定する

クライアントとのやり取りでは、以下のポイントを心がけてください：
- 経営視点と技術視点の両方からバランスの取れたアドバイスを提供する
- 具体的な事例や数値を用いて説明する
- 短期的な成果と長期的な成長のバランスを考慮する
- オープンな質問で潜在的なニーズを引き出す
- 業界特有の課題や機会に焦点を当てる

また、以下の価値提供を心がけてください：
- AWS活用によるビジネス価値の明確化
- 段階的な変革アプローチの提案
- データ駆動型の意思決定支援
- クラウドネイティブな思考による革新
- セキュリティとコンプライアンスの確保
```

## 5. 状態管理

### 5.1 エージェント状態

ITコンサルタントエージェントは以下の状態を持ちます：

1. **ビジネス理解状態**
   - 事業戦略や経営課題の把握
   - 業界環境分析
   - 企業のIT現状把握

2. **課題分析状態**
   - ITと事業の整合性分析
   - 弱点・機会・リスクの特定
   - 優先課題の整理

3. **戦略構築状態**
   - 中長期IT戦略の策定
   - 技術ロードマップの作成
   - 投資計画の立案

4. **実行計画状態**
   - アクションプランの策定
   - ステップごとのマイルストーン設定
   - リソース配分提案

5. **評価・調整状態**
   - 提案内容のレビュー
   - フィードバックに基づく調整
   - 指標設定と測定方法提案

### 5.2 状態遷移図

```
[ビジネス理解状態] --> [課題分析状態] --> [戦略構築状態] --> [実行計画状態] --> [評価・調整状態]
     ^                                                                            |
     |                                                                            |
     +----------------------------------------------------------------------------+
```

※ 状態間は必要に応じて双方向に遷移可能

### 5.3 コンテキスト管理

- **ビジネスコンテキスト**: 事業戦略、市場環境、競合情報など
- **技術コンテキスト**: 現行IT環境、技術スタック、課題点など
- **組織コンテキスト**: 組織構造、意思決定プロセス、企業文化など
- **対話コンテキスト**: 現在の対話の流れや焦点領域

## 7. エラーハンドリング

### 7.1 想定されるエラーパターン

1. **情報不足エラー**
   - 十分な事業情報が得られない場合
   - 対応: 重要な情報カテゴリを特定して質問

2. **目標不明確エラー**
   - 経営目標や方向性が不明確な場合
   - 対応: 目標設定ワークショップの提案

3. **技術知識ギャップエラー**
   - ユーザーの技術理解度が不足している場合
   - 対応: 段階的な説明と教育的アプローチ

4. **予算現実性エラー**
   - 目標と予算のバランスが取れていない場合
   - 対応: 費用対効果の高い段階的アプローチの提案

### 7.2 エラー応答例

| エラーパターン | 応答例 |
|--------------|--------|
| 情報不足エラー | 「効果的な戦略提案のために、もう少し{{情報カテゴリ}}についてお聞かせいただけますか？特に{{具体的項目}}が戦略立案の重要なインプットになります。」 |
| 目標不明確エラー | 「IT戦略を事業目標と整合させるために、御社の中期経営計画における主要KPIや優先事項について確認させてください。これにより、より的確な提案が可能になります。」 |
| 技術知識ギャップエラー | 「ご説明した技術コンセプトをより具体的にイメージいただくために、御社の業界での活用事例をいくつか共有させていただきます。こちらの方がイメージしやすいでしょうか？」 |
| 予算現実性エラー | 「ご希望の機能すべてを実現するには予算との差がございます。優先度の高い機能から段階的に実装するアプローチとして、初期フェーズでは{{重要機能}}に集中し、ROIを早期に実現する方法はいかがでしょうか？」 |

## 8. ナレッジベース参照

### 8.1 参照するナレッジカテゴリ

1. **業界別デジタル変革動向**
   - 業種ごとのDX成功事例
   - 業界固有の課題とソリューション
   - 競合ベンチマーク情報

2. **技術トレンドと展望**
   - 新興技術のビジネスインパクト
   - 技術成熟度と採用タイミング
   - 技術リスク評価基準

3. **IT投資評価フレームワーク**
   - 財務指標計算モデル
   - 非財務的効果の評価方法
   - リスク評価マトリクス

4. **クラウド戦略フレームワーク**
   - クラウド移行評価モデル
   - クラウドサービス比較
   - クラウドガバナンスモデル

5. **組織変革手法**
   - チェンジマネジメントベストプラクティス
   - IT組織構造モデル
   - デジタル人材育成アプローチ

### 8.2 参照方法

1. **業界コンテキスト参照**
   - ユーザーの業界情報に基づく関連ナレッジの検索

2. **課題ベース検索**
   - 特定された課題に関連するソリューションパターンの検索

3. **キーワード拡張検索**
   - ユーザーのキーワードを関連語彙に拡張して検索範囲を最適化

## 9. パフォーマンス指標

### 9.1 品質指標

1. **戦略的整合性**
   - 事業目標との整合度
   - 提案のフィージビリティ

2. **ユーザー満足度**
   - コンサルティング品質評価
   - インサイトの有用性評価

3. **知識の適用度**
   - 業界特有知識の活用度
   - 最新トレンドの反映度

### 9.2 効率指標

1. **ビジネス理解速度**
   - 核心課題の特定速度
   - 追加質問の効率性

2. **戦略策定時間**
   - 初期分析から提案までの時間
   - 高品質なアウトプット生成時間

3. **調整速度**
   - フィードバック反映の迅速性
   - 代替案生成のリアルタイム性

## 10. デプロイメント設計

### 10.1 実行環境

- AWS Lambda (Node.js 18.x)
- メモリ: 1536MB
- タイムアウト: 30秒
- 同時実行数: 最大50

### 10.2 スケーリング戦略

- 需要ベースの自動スケーリング
- ウォームスタート最適化によるレイテンシ削減
- リージョン間レプリケーションによる冗長性確保

### 10.3 監視設定

- CloudWatch メトリクスによるパフォーマンス監視
- レスポンス時間とクオリティスコアの追跡
- 使用状況と効果性の定期レポート生成
