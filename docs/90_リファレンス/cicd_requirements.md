# aiDev CI/CDパイプライン要件定義

## 1. 全体要件

### 1.1 基本方針
- GitHub + AWS CodePipelineによるCI/CD自動化
- インフラ、フロントエンド、バックエンドの統合的な管理
- 環境ごと（dev/stg/prod）の独立した管理
- CloudFormationによるインフラのコード化（IaC）
- 初期設定は手動実行可能とし、以降は自動化

### 1.2 デプロイ環境
- 開発環境（dev）：開発者向け、頻繁な変更を許容
- ステージング環境（stg）：テスト・検証用、本番相当の構成
- 本番環境（prod）：エンドユーザー向け、高可用性と安定性を重視

## 2. インフラストラクチャ要件

### 2.1 IaC（Infrastructure as Code）
- AWSリソースはすべてCloudFormationで定義
- AWSサービスごとに独立したテンプレートファイルを作成
- 環境間で共通の設定はパラメータとして切り出し
- ネストされたスタック構造で保守性を向上
- 環境固有の設定値はパラメータファイルで管理

### 2.2 リソース構成
- ネットワーク（VPC、サブネット、セキュリティグループ）
- コンピューティング（Lambda、ECS/Fargate）
- ストレージ（S3、DynamoDB）
- API（API Gateway）
- 認証（Cognito）
- CDN（CloudFront）
- モニタリング（CloudWatch）
- セキュリティ（IAM、WAF）

### 2.3 リソース命名規則
- 環境を接頭辞として使用（例：`dev-aidev-resource`）
- リソースタイプを含める（例：`dev-aidev-lambda-hello`）
- 一貫した命名規則でリソースの識別を容易に

## 3. CI/CDパイプライン要件

### 3.1 初期設定
- CodePipeline初期設定用のCloudFormationテンプレート
- GitHubとの連携はGUIを通じたOAuth認証を使用
- IAMロールと権限の最小権限原則に基づく設定
- 初期設定実行のための手順書作成

### 3.2 パイプラインの構成
- ソース段階：GitHubリポジトリからのコード取得
- ビルド段階：CodeBuildによるビルド、テスト実行
- 承認段階：環境デプロイ前の手動承認（stg/prodのみ）
- デプロイ段階：CloudFormationによるインフラ/アプリのデプロイ

### 3.3 環境分離
- 環境ごとに独立したパイプライン
- 環境固有のパラメータファイル
- dev環境は自動デプロイ、stg/prod環境は承認後デプロイ

### 3.4 通知とフィードバック
- パイプライン実行状態のSNS通知
- ビルド/デプロイ失敗時のアラート
- デプロイ完了時の通知

## 4. フロントエンド CI/CD要件

### 4.1 ビルドプロセス
- Node.jsによるReactアプリケーションのビルド
- 環境変数の適切な注入
- 静的アセットの最適化（圧縮、バンドル）

### 4.2 テスト
- 単体テスト（Jest）
- E2Eテスト（Cypress）
- コードの品質チェック（ESLint、Prettier）

### 4.3 デプロイ
- S3バケットへの静的ファイルのアップロード
- CloudFrontキャッシュの無効化
- デプロイバージョンの管理

## 5. バックエンド CI/CD要件

### 5.1 ビルドプロセス
- Lambda関数のビルドとパッケージング
- 依存関係の管理
- 環境変数の設定

### 5.2 テスト
- Lambda関数の単体テスト
- APIの統合テスト
- セキュリティチェック

### 5.3 デプロイ
- Lambda関数の更新
- API Gatewayの設定変更
- DynamoDBなどのデータストアの更新

## 6. セキュリティ要件

### 6.1 認証・認可
- IAMロールの最小権限原則
- クロスアカウントアクセス権限の管理
- シークレット管理の適切な運用

### 6.2 コードセキュリティ
- 依存関係の脆弱性スキャン
- SAST（静的アプリケーションセキュリティテスト）
- コードレビュープロセスの強制

### 6.3 インフラセキュリティ
- ネットワークセキュリティの確保（VPC、セキュリティグループ）
- 暗号化（保存データ、転送中データ）
- コンプライアンスチェック（AWS Config Rules）

## 7. 監視・運用要件

### 7.1 ログ管理
- 集中ログ管理
- ログの構造化
- ログ保持期間の設定

### 7.2 アラート
- 異常検知とアラート通知
- パフォーマンス指標のモニタリング
- 自動復旧メカニズム

### 7.3 デプロイ監視
- デプロイ結果の自動検証
- カナリアデプロイによる段階的リリース（将来拡張）
- ロールバック手順の自動化
