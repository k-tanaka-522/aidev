# aiDev プロジェクト引継ぎファイル

## プロジェクト概要

aiDevは、対話型AIを活用した「AI開発支援窓口（プリセールスエージェント）」を構築するプロジェクトです。AWS Bedrockを活用し、ユーザーとの対話を通じて開発の相談やAWS環境の設計・構築・改善の支援を行います。

   **CI/CD環境セットアップの詳細手順（GitHub連携版）**:
   1. ターミナル（WSLまたはWindows Terminal）を開く
   2. 以下のコマンドを実行:
      ```bash
      cd /mnt/c/dev2/aiDev
      ./src/infra-init/deploy-cicd.sh -e dev -r ap-northeast-1 -o k-tanaka-522 -g aidev -b main -c <CloudFrontディストリビューションID>
      ```
      ここで:
      - `-o k-tanaka-522`: GitHub所有者名
      - `-g aidev`: GitHubリポジトリ名
      - `-b main`: GitHubブランチ名
      - `-c <ID>`: CloudFrontディストリビューションID (オプション)
   3. プロンプトの確認に `y` と入力
   4. スクリプトの完了を待つ
   
   **注意事項**:
   - このスクリプトは冪等性を考慮して設計されており、誤って複数回実行しても問題ありません
   - 既存のスタックが検出された場合は自動的に更新（update-stack）モードで実行されます
   - 実行のたびに詳細なログが表示されるため、変更内容を確認できます
   - CloudFormationスタックの作成/更新状態はAWSコンソールでも確認できます
   - GitHub接続は初回作成後、AWSコンソールでの手動承認が必要です
   
   **GitHub接続の手動承認手順**:
   1. スクリプト実行後、出力される「GitHub接続ARN」と「GitHub接続ステータス」を確認
   2. AWS Management Consoleにログイン
   3. 日本語表示の場合: 「CodePipeline」に移動し、左側メニューから「設定」を選択して「接続」タブを確認
      英語表示の場合: 「Developer Tools」>「Settings」>「Connections」に移動
   4. 作成された接続（名前: `aiDev-dev-github-connection`）を選択
   5. 「保留中」/「Pending」状態の接続に対して「保留中の接続を更新」/「Update pending connection」ボタンをクリック
   6. 表示されるダイアログでGitHubアカウントにログインし、接続を承認
   7. 接続ステータスが「利用可能」/「Available」になったことを確認

   **GitHub接続後の確認手順**:
   1. ソースコードに変更を加え、GitHubリポジトリにプッシュ
   2. AWS Management Consoleの「CodePipeline」でパイプラインが自動的に起動したことを確認
   3. パイプラインの各ステージ（ソース、ビルド、デプロイ）が正常に完了することを確認
   4. 特にCloudFront無効化ステップが正常に動作することを確認
